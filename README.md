# Claude Code Coaching Materials

Enterprise Claude Code onboarding that gets a senior engineer from zero to productive in one 90-minute session. Built and tested in live coaching engagements with enterprise technical teams.

## What's here

**`claude-md-builder.md`**: a Claude Code skill that replaces the default `/init` with a 5-question interview about role, daily workflow, tools, writing style, and compliance rules, then writes a CLAUDE.md using the user's exact language. Used as the first skill every new coaching client installs.

**`coaching-getting-started.md`**: a compliance-first getting-started guide for enterprise engineers. Covers installation (Homebrew cask, native installer, enterprise SSO), daily Glean-to-Claude routine, project structure, CLAUDE.md as living memory, skills, token efficiency, and troubleshooting. Front-loads the enterprise data-handling rules most Claude Code tutorials skip.

## Why this exists

Most Claude Code onboarding assumes the user is a developer working on their own codebase. Enterprise adopters are often non-developer knowledge workers, or developers working under compliance constraints their personal workflows never had. These two artifacts replace generic onboarding with enterprise-context-aware onboarding: compliance rules first, tailored CLAUDE.md driven by interview, Glean as the bridge between the user's real work data and Claude Code.
